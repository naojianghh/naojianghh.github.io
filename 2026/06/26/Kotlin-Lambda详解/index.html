Kotlin Lambda详解

2026/6/26 Android Android Kotlin

Lambda 表达式是 Kotlin 最强大的语言特性之一。它让代码更加简洁、函数式编程成为可能，也是集合操作、协程等高级特性的基石。本文从零开始，带你彻底掌握 Kotlin 的 Lambda。

什么是 Lambda？

Lambda 本质上就是一个匿名函数——没有名字、可以直接传递的函数。它可以作为参数传给另一个函数，也可以赋值给变量，甚至可以作为一个函数的返回值。

// 普通函数
fun add(a: Int, b: Int): Int {
    return a + b
}

// 等价的 Lambda 表达式
val add: (Int, Int) -> Int = { a, b -> a + b }

// 调用方式完全相同
add(1, 2)  // 3

为什么需要 Lambda？

传统的面向对象语言中，想要传递一个行为，通常需要创建一个匿名内部类：

// Java 风格：繁琐的匿名内部类
button.setOnClickListener(object : View.OnClickListener {
    override fun onClick(v: View?) {
        println("按钮被点击了")
    }
})

// Kotlin Lambda：一行搞定
button.setOnClickListener { println("按钮被点击了") }

Lambda 让”传递行为”变得像”传递数据”一样简单。

Lambda 语法全解

基本语法结构

{ 参数名: 参数类型 -> 函数体 }
  • 花括号 {} 包裹整个 Lambda
  • -> 左边是参数列表，右边是函数体
  • 最后一行自动作为返回值（不需要 return）
// 完整写法
val sum: (Int, Int) -> Int = { x: Int, y: Int -> x + y }

// 类型推断：类型声明在变量上，Lambda 内可省略类型
val sum2: (Int, Int) -> Int = { x, y -> x + y }

// 反过来：类型写在 Lambda 内，变量声明可省略
val sum3 = { x: Int, y: Int -> x + y }

不同类型推断方式对比

// 方式一：变量声明类型，Lambda 内推断
val add: (Int, Int) -> Int = { a, b -> a + b }

// 方式二：Lambda 内声明类型，变量推断
val add2 = { a: Int, b: Int -> a + b }

// 方式三：两者都不声明 → 编译错误！
// val add3 = { a, b -> a + b }  // 错误：无法推断类型

// 方式四：有接收者时从上下文推断
listOf(1, 2, 3).map { it * 2 }  // it 自动推断为 Int

单参数简写：it

当 Lambda 只有一个参数时，可以省略参数声明，用 it 代替：

val numbers = listOf(1, 2, 3, 4, 5)

// 完整写法
numbers.filter { number -> number > 3 }

// it 简写
numbers.filter { it > 3 }

// 但参数含义不明确时不建议用 it，影响可读性
people.filter { it.age > 18 }  // 还行
people.groupBy { it }          // 这是什么？建议用命名参数

无参数 Lambda

// 无参数时，-> 可以省略
val sayHello: () -> Unit = { println("Hello") }

// 或者显式写出 ->
val sayHello2: () -> Unit = { -> println("Hello") }

// 调用
sayHello()

多行 Lambda

val calculate: (Int, Int) -> Int = { a, b ->
    println("计算 $a + $b")
    val result = a + b
    result  // 最后一行是返回值，不要写 return
}

函数类型

Lambda 可以赋值给变量、作为参数传递，这背后是 Kotlin 的函数类型（Function Type）机制。

函数类型的写法

// 基础形式
(参数类型列表) -> 返回值类型

// 举例
() -> Unit                    // 无参无返回值
(Int, String) -> Boolean      // 两个参数，返回 Boolean
(Int) -> (Int) -> Boolean     // 返回一个函数（柯里化）

声明函数类型的变量

// 声明一个函数类型的变量
var action: (String) -> Unit = { msg -> println(msg) }

// 可以重新赋值（只要类型匹配）
action = { println("收到：$it") }

// 调用
action("Hello")  // 输出：收到：Hello

函数类型作为参数

// 接收一个 String → Int 的函数作为参数
fun processText(text: String, transform: (String) -> Int): Int {
    return transform(text)
}

// 调用方式
val length = processText("Kotlin") { it.length }           // 6
val hash = processText("Kotlin") { it.hashCode() }          // 具体值
val count = processText("Hello World") { it.split(" ").size } // 2

函数类型作为返回值

// 返回一个函数
fun createMultiplier(factor: Int): (Int) -> Int {
    return { number -> number * factor }
}

val double = createMultiplier(2)
val triple = createMultiplier(3)

println(double(5))  // 10
println(triple(5))  // 15

尾随 Lambda 语法

这是 Kotlin 最实用的语法糖之一：当函数的最后一个参数是 Lambda 时，Lambda 可以写在括号外面。

// 标准调用
list.fold(0, { acc, item -> acc + item })

// 尾随 Lambda：更干净
list.fold(0) { acc, item -> acc + item }

常见场景

// 如果 Lambda 是唯一参数，连括号都可以省略
list.forEach({ item -> println(item) })
list.forEach { println(it) }  // 括号也不需要

// with、apply 等作用域函数
val user = User().apply {
    name = "张三"
    age = 18
}

// 协程
scope.launch {
    delay(1000)
    println("Hello")
}

多个参数时的位置

fun repeat(times: Int, action: (Int) -> Unit) {
    for (i in 0 until times) action(i)
}

// Lambda 在前：必须用括号包裹
repeat({ println(it) }, 3)  // 不推荐

// Lambda 在最后：可以用尾随语法
repeat(3) { println(it) }   // 推荐

高阶函数

高阶函数是指接收函数作为参数，或者返回一个函数的函数。

map、filter、fold 等集合操作

这些是最常用的高阶函数：

val numbers = listOf(1, 2, 3, 4, 5, 6)

// filter：过滤
val evens = numbers.filter { it % 2 == 0 }  // [2, 4, 6]

// map：转换
val squares = numbers.map { it * it }  // [1, 4, 9, 16, 25, 36]

// fold：累加（带初始值）
val sum = numbers.fold(0) { acc, n -> acc + n }  // 21

// reduce：累加（无初始值，用第一个元素作为初始值）
val sum2 = numbers.reduce { acc, n -> acc + n }  // 21

// any / all / none：条件判断
val hasBig = numbers.any { it > 5 }   // true
val allSmall = numbers.all { it < 10 } // true
val noneBig = numbers.none { it > 10 } // true

链式调用

data class Person(val name: String, val age: Int)

val people = listOf(
    Person("张三", 18),
    Person("李四", 25),
    Person("王五", 30)
)

// 链式操作：找出成年人，取名字，转为大写，排序
val result = people
    .filter { it.age >= 18 }
    .map { it.name.uppercase() }
    .sorted()
// [李四, 王五, 张三]

flatMap 展平映射

flatMap 是 map + flatten 的组合：先用 map 把每个元素转成一个集合，再把这些小集合拍扁合并成一个大集合。

一个典型场景：有一批用户，要取出每个人的所有订单：

data class User(val name: String, val orders: List<String>)

val users = listOf(
    User("张三", listOf("订单A", "订单B")),
    User("李四", listOf("订单C")),
    User("王五", listOf("订单D", "订单E", "订单F"))
)

// 取出所有用户的订单，合并成一个列表
val allOrders = users.flatMap { it.orders }
// ["订单A", "订单B", "订单C", "订单D", "订单E", "订单F"]

执行过程拆解：

val nested = listOf(listOf(1, 2), listOf(3, 4), listOf(5, 6))

// 第一步：map → 每个元素变为自身的平方
nested.map { subList -> subList.map { n -> n * n } }
// 结果：[[1, 4], [9, 16], [25, 36]]  ← 还是嵌套的列表

// 第二步：flatMap = map + flatten，一步到位
nested.flatMap { subList -> subList.map { n -> n * n } }
// 结果：[1, 4, 9, 16, 25, 36]  ← 已拍扁

flatMap vs map 的区别：

val nested = listOf(listOf(1, 2), listOf(3, 4))

nested.map { it.map { it * 2 } }    // [[2, 4], [6, 8]]  嵌套结构
nested.flatMap { it.map { it * 2 } } // [2, 4, 6, 8]      扁平结构

groupBy 分组

val grouped = people.groupBy { if (it.age >= 18) "成年" else "未成年" }
// {成年=[Person(张三,18), Person(李四,25), Person(王五,30)], 未成年=[]}

Lambda 与闭包

Lambda 可以捕获外部变量，形成闭包。在 Kotlin 中捕获的是变量引用，而不是值：

var counter = 0

val increment: () -> Unit = { counter++ }

increment()
increment()
println(counter)  // 2 —— 外部变量被同步修改

对比 Java 的差异

Java 的匿名内部类只能捕获 final（或 effectively final）变量：

// Kotlin：可变变量也可以捕获
var count = 0
val action = { count++ }  // 完全没问题

// Java 等价的匿名内部类做不到（需要 AtomicInteger 等手段）

Lambda 与 Receiver（带接收者的 Lambda）

这是 Kotlin 独有的强大特性。带接收者的 Lambda 可以直接访问接收者的成员，不需要 it 或显式引用。

语法

接收者类型.() -> 返回值类型

实际应用：作用域函数

// apply：接收者是调用对象本身，返回对象自身
val user = User().apply {
    name = "张三"     // 直接访问 User 的属性，不需要 user.name
    age = 18
}

// also：接收者也是调用对象，但用 it 引用
val user2 = User().also { user ->
    println("创建了用户：${user.name}")
}

// let：接收者是调用对象，用 it 引用，返回 Lambda 结果
val name = user.let { it.name }

// run：接收者是调用对象，直接访问，返回 Lambda 结果
val greeting = user.run {
    "Hello, I'm $name, $age years old"
}

// with：非扩展函数，第一个参数是接收者
val info = with(user) {
    "name=$name, age=$age"
}

Type-Safe Builder（类型安全构建器）

带接收者的 Lambda 最经典的用途是 DSL 构建：

// 构建 HTML
html {
    head {
        title { +"My Page" }
    }
    body {
        h1 { +"Welcome" }
        p { +"This is a paragraph." }
    }
}

// 构建 Android 布局（Anko 风格）
verticalLayout {
    textView {
        text = "Hello"
        textSize = 16f
    }
    button {
        text = "Click Me"
        onClick { showToast("Clicked!") }
    }
}

实现原理很简单：

class HTML {
    fun body(init: Body.() -> Unit) {
        val body = Body()
        body.init()  // 在 Body 的上下文中执行 Lambda
        children.add(body)
    }
}

// 调用时
html {  // this 是 HTML
    body {  // this 是 Body，可以直接调用 Body 的方法
        // ...
    }
}

SAM 转换（Single Abstract Method）

当一个 Java 接口只有一个抽象方法时，Kotlin 可以用 Lambda 代替匿名内部类。

// Java 接口
interface OnClickListener {
    void onClick(View v);
}

// Kotlin 匿名内部类写法
button.setOnClickListener(object : View.OnClickListener {
    override fun onClick(v: View) {
        println("clicked")
    }
})

// SAM 转换：用 Lambda 代替
button.setOnClickListener { v -> println("clicked") }

// 更常见的：参数不用时可以省略（包括 ->）
button.setOnClickListener { println("clicked") }

为什么 v 可以省略？ Kotlin 的 Lambda 规则：如果参数没有被使用，可以连参数声明和 -> 一起省略，变成无参数的 Lambda 写法。这在实际开发中非常常见——大多数点击事件都不需要用到被点击的 View。

SAM 转换的规则

// 1. 必须是 Java 接口（Kotlin 接口不支持自动 SAM，但可以用 fun interface）
// 2. 接口必须有且只有一个抽象方法
// 3. Lambda 参数类型和返回值类型必须匹配

// Runnable — 典型 SAM
val runnable = Runnable { println("running") }

// Callable
val callable = Callable { 42 }

Kotlin 的 fun interface（函数式接口）

Kotlin 1.4+ 引入了 fun interface，让 Kotlin 接口也支持 SAM：

fun interface Transformer<T> {
    fun transform(input: T): T
}

// SAM 转换
val upper: Transformer<String> = Transformer { it.uppercase() }
println(upper.transform("hello"))  // HELLO

// 普通调用
val list = listOf("a", "b", "c")
val mapped = list.map(Transformer { it.uppercase() })

集合序列与惰性求值

链式调用高阶函数时，每个操作都会创建一个新的集合。序列（Sequence）可以避免这一点：

val numbers = (1..1_000_000).toList()

// 急切求值：每一步都产生中间集合（内存杀手）
numbers
    .filter { it % 2 == 0 }     // 50 万个元素
    .map { it * it }             // 50 万个元素
    .take(10)                    // 取前 10 个

// 惰性求值：最终只计算 20 个元素
numbers.asSequence()
    .filter { it % 2 == 0 }
    .map { it * it }
    .take(10)
    .toList()

规则：

  • 数据量小（< 1000）：直接用集合操作，简单高效
  • 数据量大或链式操作多：优先 asSequence()，避免中间集合的内存开销
  • 无限序列也能处理：generateSequence(0) { it + 1 }.take(5).toList() → [0, 1, 2, 3, 4]

常见误区与最佳实践

误区一：在 Lambda 中使用 return

// 错误：return 会直接从外层函数返回（非局部返回）
fun processList(items: List<Int>) {
    items.forEach {
        if (it < 0) return  // 危险！直接从 processList 返回
        println(it)
    }
}

// 正确：用标签限定返回范围
fun processList(items: List<Int>) {
    items.forEach {
        if (it < 0) return@forEach  // 只跳过当前元素
        println(it)
    }
}

误区二：忽略可读性，滥用 it

// 不推荐：不清晰
list.map { it.first }.filter { it.count() > 3 }

// 推荐：命名参数
list.map { user -> user.name }.filter { name -> name.count() > 3 }

误区三：嵌套过深

// 不推荐
button.setOnClickListener {
    scope.launch {
        val data = withContext(Dispatchers.IO) {
            api.fetchData().also { cache.save(it) }
        }
        updateUI(data)
    }
}

// 推荐：提取为独立函数
button.setOnClickListener { onButtonClick() }

private fun onButtonClick() {
    scope.launch {
        val data = fetchAndCache()
        updateUI(data)
    }
}

private suspend fun fetchAndCache() = withContext(Dispatchers.IO) {
    api.fetchData().also { cache.save(it) }
}

误区四：误用作用域函数

函数 接收者引用 返回值 典型场景
let it Lambda 结果 可空链式调用、局部变量作用域
run this Lambda 结果 对象配置并计算结果
with this Lambda 结果 对同一对象多次操作
apply this 对象自身 对象初始化配置
also it 对象自身 附加操作（日志、校验等） 
// apply：初始化对象
val user = User().apply {
    name = "张三"
    age = 18
}

// let：处理可空对象
val length = nullableString?.let { it.length } ?: 0

// also：附加操作
val result = api.fetchData().also { Log.d("API", "返回数据：$it") }

实战场景

场景一：集合操作的终极组合

data class Order(val id: Int, val product: String, val price: Double, val count: Int)

val orders = listOf(
    Order(1, "键盘", 299.0, 2),
    Order(2, "鼠标", 99.0, 5),
    Order(3, "显示器", 1999.0, 1),
    Order(4, "键盘", 199.0, 3)
)

// 需求：计算所有 "键盘" 的总销售额
val keyboardRevenue = orders
    .asSequence()
    .filter { it.product == "键盘" }
    .map { it.price * it.count }
    .sum()
// 299*2 + 199*3 = 1195.0

场景二：回调模式

// 定义带回调的函数
fun fetchData(onSuccess: (String) -> Unit, onError: (Throwable) -> Unit) {
    try {
        val data = readFromNetwork()
        onSuccess(data)
    } catch (e: Exception) {
        onError(e)
    }
}

// 调用
fetchData(
    onSuccess = { data -> println("成功：$data") },
    onError = { e -> println("失败：${e.message}") }
)

场景三：策略模式

// 不同排序策略
val byName: Comparator<Person> = compareBy { it.name }
val byAge: Comparator<Person> = compareBy { it.age }

people.sortedWith(byAge)   // 按年龄排序
people.sortedWith(byName)  // 按姓名排序

// 组合策略
people.sortedWith(compareBy<Person> { it.age }.thenBy { it.name })

场景四：高阶函数封装

// 封装一个测量执行时间的工具函数
inline fun <T> measureTime(tag: String = "执行", block: () -> T): T {
    val start = System.currentTimeMillis()
    val result = block()
    val duration = System.currentTimeMillis() - start
    println("$tag 耗时：${duration}ms")
    return result
}

// 使用
val data = measureTime("网络请求") {
    api.fetchData()
}
// 输出：网络请求 耗时：235ms

注意 inline 关键字：对于高阶函数，建议加上 inline 以避免每次调用都创建 Lambda 对象，提升性能。

总结

Kotlin Lambda 的核心知识体系：

  1. Lambda 就是匿名函数：用 {} 包裹，-> 分隔参数和函数体，末行自动返回
  2. 函数类型：(参数) -> 返回值，让函数像数据一样传递
  3. 尾随 Lambda：最后一个参数是 Lambda 时写在括号外，代码更清晰
  4. 高阶函数：map、filter、fold、flatMap 等让集合操作变得优雅
  5. 带接收者的 Lambda：apply、run、with 等作用域函数的核心机制
  6. SAM 转换：Lambda 替换 Java 单方法接口的匿名内部类
  7. 闭包：Lambda 捕获外部变量引用，可变变量也能捕获

记住三点：

  • 简单操作别怕写 Lambda，越写越顺手
  • 复杂 Lambda 提取为函数，保持代码可读性
  • 高阶函数加 inline，性能更好
