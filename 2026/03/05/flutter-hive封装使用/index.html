Flutter Hive 封装使用

2026/3/5 移动端开发 flutter 工具

Hive 是 Flutter 中一个轻量级的 NoSQL 数据库，专为移动应用和桌面应用设计。

Hive 简介

它具有以下特点：

  • 高性能：比 SQLite 更快，特别是在读取操作上
  • 轻量级：核心库很小，适合移动应用
  • 易于使用：API 简洁明了
  • 跨平台：支持 iOS、Android、Web、macOS、Windows、Linux
  • 不需要原生依赖：纯 Dart 实现，不需要原生代码

Hive 安装与配置

1. 添加依赖

在 pubspec.yaml 文件中添加 Hive 相关依赖：

dependencies:
  hive: ^2.2.3
  hive_flutter: ^1.1.0
  path_provider: ^2.0.11

dev_dependencies:
  hive_generator: ^2.0.0
  build_runner: ^2.3.3

2. 初始化 Hive

在应用启动时初始化 Hive：

import 'package:hive/hive.dart';
import 'package:hive_flutter/hive_flutter.dart';
import 'package:path_provider/path_provider.dart';

void main() async {
  // 初始化 Hive Flutter
  await Hive.initFlutter();
  
  // 注册适配器
  // await Hive.registerAdapter(UserAdapter());
  
  // 打开盒子
  // await Hive.openBox('users');
  
  runApp(MyApp());
}

Hive 基本使用

1. 创建模型类

对于复杂数据类型，需要创建模型类并生成适配器：

import 'package:hive/hive.dart';

part 'user.g.dart';

@HiveType(typeId: 0)
class User {
  @HiveField(0)
  String id;
  
  @HiveField(1)
  String name;
  
  @HiveField(2)
  int age;
  
  @HiveField(3)
  bool isActive;
  
  User({
    required this.id,
    required this.name,
    required this.age,
    required this.isActive,
  });
}

2. 生成适配器

运行以下命令生成适配器：

flutter packages pub run build_runner build

3. 基本操作

// 打开盒子
final box = await Hive.openBox('users');

// 添加数据
final user = User(
  id: '1',
  name: 'John',
  age: 25,
  isActive: true,
);
await box.put('user1', user);

// 获取数据
final savedUser = box.get('user1');
print('Name: ${savedUser.name}');

// 更新数据
user.age = 26;
await box.put('user1', user);

// 删除数据
await box.delete('user1');

// 清空盒子
await box.clear();

// 关闭盒子
await box.close();

Hive 封装使用

为了方便使用，我们可以对 Hive 进行封装，创建一个数据库管理工具类。

1. 创建 Hive 管理类

import 'package:hive/hive.dart';
import 'package:path_provider/path_provider.dart';

class HiveManager {
  static final HiveManager _instance = HiveManager._internal();
  factory HiveManager() => _instance;
  
  Map<String, Box> _boxes = {};
  
  HiveManager._internal();
  
  // 初始化 Hive
  Future<void> init() async {
    try {
      // 获取应用文档目录
      final directory = await getApplicationDocumentsDirectory();
      Hive.init(directory.path);
      
      // 注册适配器
      // 这里可以注册所有的适配器
      // await Hive.registerAdapter(UserAdapter());
      // await Hive.registerAdapter(ProductAdapter());
    } catch (e) {
      print('Hive 初始化失败: $e');
      rethrow;
    }
  }
  
  // 打开盒子
  Future<Box> openBox(String boxName) async {
    try {
      if (_boxes.containsKey(boxName)) {
        return _boxes[boxName]!;
      }
      
      final box = await Hive.openBox(boxName);
      _boxes[boxName] = box;
      return box;
    } catch (e) {
      print('打开盒子失败: $e');
      rethrow;
    }
  }
  
  // 获取盒子
  Box getBox(String boxName) {
    if (!_boxes.containsKey(boxName)) {
      throw Exception('盒子 $boxName 未打开');
    }
    return _boxes[boxName]!;
  }
  
  // 关闭盒子
  Future<void> closeBox(String boxName) async {
    try {
      if (_boxes.containsKey(boxName)) {
        await _boxes[boxName]!.close();
        _boxes.remove(boxName);
      }
    } catch (e) {
      print('关闭盒子失败: $e');
      rethrow;
    }
  }
  
  // 关闭所有盒子
  Future<void> closeAllBoxes() async {
    try {
      for (final boxName in _boxes.keys) {
        await _boxes[boxName]!.close();
      }
      _boxes.clear();
    } catch (e) {
      print('关闭所有盒子失败: $e');
      rethrow;
    }
  }
  
  // 删除盒子
  Future<void> deleteBox(String boxName) async {
    try {
      if (_boxes.containsKey(boxName)) {
        await _boxes[boxName]!.close();
        _boxes.remove(boxName);
      }
      await Hive.deleteBoxFromDisk(boxName);
    } catch (e) {
      print('删除盒子失败: $e');
      rethrow;
    }
  }
}

2. 创建数据访问类

为了更好地管理数据，我们可以为每个模型创建数据访问类：

import 'package:hive/hive.dart';
import '../models/user.dart';
import '../services/hive_manager.dart';

class UserRepository {
  late Box<User> _userBox;
  
  Future<void> init() async {
    _userBox = await HiveManager().openBox('users');
  }
  
  // 添加用户
  Future<void> addUser(User user) async {
    await _userBox.put(user.id, user);
  }
  
  // 获取用户
  User? getUser(String id) {
    return _userBox.get(id);
  }
  
  // 获取所有用户
  List<User> getAllUsers() {
    return _userBox.values.toList();
  }
  
  // 更新用户
  Future<void> updateUser(User user) async {
    await _userBox.put(user.id, user);
  }
  
  // 删除用户
  Future<void> deleteUser(String id) async {
    await _userBox.delete(id);
  }
  
  // 清空所有用户
  Future<void> clearUsers() async {
    await _userBox.clear();
  }
  
  // 关闭盒子
  Future<void> close() async {
    await HiveManager().closeBox('users');
  }
}

3. 创建服务层

为了进一步封装业务逻辑，我们可以创建服务层：

import '../repositories/user_repository.dart';
import '../models/user.dart';

class UserService {
  final UserRepository _userRepository = UserRepository();
  
  Future<void> init() async {
    await _userRepository.init();
  }
  
  // 创建用户
  Future<void> createUser(User user) async {
    await _userRepository.addUser(user);
  }
  
  // 获取用户详情
  User? getUser(String id) {
    return _userRepository.getUser(id);
  }
  
  // 获取用户列表
  List<User> getUsers() {
    return _userRepository.getAllUsers();
  }
  
  // 更新用户信息
  Future<void> updateUser(User user) async {
    await _userRepository.updateUser(user);
  }
  
  // 删除用户
  Future<void> deleteUser(String id) async {
    await _userRepository.deleteUser(id);
  }
  
  // 搜索用户
  List<User> searchUsers(String keyword) {
    final users = _userRepository.getAllUsers();
    return users.where((user) => 
      user.name.toLowerCase().contains(keyword.toLowerCase())
    ).toList();
  }
  
  // 关闭服务
  Future<void> close() async {
    await _userRepository.close();
  }
}

完整示例

1. 应用初始化

import 'package:flutter/material.dart';
import 'package:hive_flutter/hive_flutter.dart';
import 'services/hive_manager.dart';
import 'services/user_service.dart';
import 'models/user.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  
  // 初始化 Hive
  await HiveManager().init();
  
  // 注册适配器
  Hive.registerAdapter(UserAdapter());
  
  runApp(MyApp());
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'Hive 示例',
      theme: ThemeData(primarySwatch: Colors.blue),
      home: HomePage(),
    );
  }
}

2. 主页面

import 'package:flutter/material.dart';
import 'services/user_service.dart';
import 'models/user.dart';

class HomePage extends StatefulWidget {
  @override
  _HomePageState createState() => _HomePageState();
}

class _HomePageState extends State<HomePage> {
  final UserService _userService = UserService();
  List<User> _users = [];
  TextEditingController _nameController = TextEditingController();
  TextEditingController _ageController = TextEditingController();
  
  @override
  void initState() {
    super.initState();
    _init();
  }
  
  Future<void> _init() async {
    await _userService.init();
    _loadUsers();
  }
  
  void _loadUsers() {
    setState(() {
      _users = _userService.getUsers();
    });
  }
  
  void _addUser() async {
    final name = _nameController.text;
    final age = int.tryParse(_ageController.text) ?? 0;
    
    if (name.isNotEmpty) {
      final user = User(
        id: DateTime.now().millisecondsSinceEpoch.toString(),
        name: name,
        age: age,
        isActive: true,
      );
      
      await _userService.createUser(user);
      _loadUsers();
      _nameController.clear();
      _ageController.clear();
    }
  }
  
  void _deleteUser(String id) async {
    await _userService.deleteUser(id);
    _loadUsers();
  }
  
  @override
  void dispose() {
    _userService.close();
    _nameController.dispose();
    _ageController.dispose();
    super.dispose();
  }
  
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: Text('Hive 示例')),
      body: Padding(
        padding: EdgeInsets.all(16.0),
        child: Column(
          children: [
            TextField(
              controller: _nameController,
              decoration: InputDecoration(labelText: '姓名'),
            ),
            TextField(
              controller: _ageController,
              decoration: InputDecoration(labelText: '年龄'),
              keyboardType: TextInputType.number,
            ),
            ElevatedButton(
              onPressed: _addUser,
              child: Text('添加用户'),
            ),
            Expanded(
              child: ListView.builder(
                itemCount: _users.length,
                itemBuilder: (context, index) {
                  final user = _users[index];
                  return ListTile(
                    title: Text(user.name),
                    subtitle: Text('年龄: ${user.age}, 状态: ${user.isActive ? '活跃' : '非活跃'}'),
                    trailing: IconButton(
                      icon: Icon(Icons.delete),
                      onPressed: () => _deleteUser(user.id),
                    ),
                  );
                },
              ),
            ),
          ],
        ),
      ),
    );
  }
}

Hive 最佳实践

1. 数据模型设计

  • 使用类型ID：为每个模型类分配唯一的 typeId
  • 字段顺序：保持字段顺序一致，避免序列化问题
  • 版本控制：当模型结构变化时，考虑使用 Hive 的版本控制

2. 性能优化

  • 批量操作：使用 putAll、deleteAll 等批量操作减少磁盘 I/O
  • 索引：对于频繁查询的字段，考虑创建索引
  • 盒子管理：合理管理盒子的打开和关闭，避免资源泄漏

3. 错误处理

  • 异常捕获：在所有 Hive 操作中添加异常捕获
  • 备份策略：定期备份重要数据
  • 恢复机制：实现数据恢复机制，处理盒子损坏的情况

4. 安全性

  • 加密：对于敏感数据，使用 Hive 的加密功能
  • 权限控制：在移动应用中，确保数据文件的权限设置正确
  • 数据清理：及时清理不再需要的数据

总结

Hive 是 Flutter 中一个强大而灵活的本地存储解决方案，通过合理的封装，我们可以更方便地使用它来管理应用数据。本文介绍了 Hive 的基本使用方法和封装技巧，希望对你有所帮助。

通过封装，我们可以：

  1. 简化代码：减少重复代码，提高代码可读性
  2. 统一管理：集中管理数据库操作，便于维护
  3. 提高可测试性：便于单元测试和集成测试
  4. 增强扩展性：易于添加新功能和修改现有功能

在实际应用中，你可以根据具体需求进一步扩展和优化这个封装方案，以满足你的业务需求。
