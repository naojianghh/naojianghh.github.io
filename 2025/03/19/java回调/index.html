java回调

2025/3/19 学习 java

java回调入门学习。

概念

回调是一种双向调用机制，简单来说，就是 A 类调用 B 类的方法 C，B 类在执行方法 C 的过程中，在某个特定时刻调用 A 类的方法 D，这里方法 D 就是回调方法。

实现方式

Java 实现回调通常需要以下几个步骤：

  1. 定义回调接口：声明回调方法，该方法将在合适的时机被调用。
  2. 实现回调接口：创建一个类来实现这个回调接口，并实现其中的回调方法。
  3. 在被调用类中使用回调接口：被调用类接收一个实现了回调接口的对象作为参数，并在适当的时候调用回调方法。

示例代码

// 1. 定义回调接口
interface TaskCompletionCallback {
    void onTaskCompleted(String result);
}

// 2. 实现回调接口
class TaskCaller implements TaskCompletionCallback {
    @Override
    public void onTaskCompleted(String result) {
        System.out.println("任务已完成，结果是: " + result);
    }
}

// 3. 在被调用类中使用回调接口
class TaskExecutor {
    private TaskCompletionCallback callback;

    public TaskExecutor(TaskCompletionCallback callback) {
        this.callback = callback;
    }

    public void executeTask() {
        // 模拟任务执行
        System.out.println("正在执行任务...");
        try {
            Thread.sleep(2000); // 模拟耗时操作
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        String result = "任务成功完成";
        // 任务完成后调用回调方法
        callback.onTaskCompleted(result);
    }
}

// 测试代码
public class CallbackExample {
    public static void main(String[] args) {
        TaskCaller caller = new TaskCaller();
        TaskExecutor executor = new TaskExecutor(caller);
        executor.executeTask();
    }
}

代码解释

  1. **定义回调接口 TaskCompletionCallback**：该接口中定义了一个 onTaskCompleted 方法，用于在任务完成时通知调用者。
  2. **实现回调接口 TaskCaller**：TaskCaller 类实现了 TaskCompletionCallback 接口，并实现了 onTaskCompleted 方法，当任务完成时，会打印任务结果。
  3. **被调用类 TaskExecutor**：TaskExecutor 类接收一个 TaskCompletionCallback 类型的对象作为参数，并在 executeTask 方法中模拟任务执行。任务完成后，调用回调对象的 onTaskCompleted 方法。
  4. 测试代码：在 main 方法中，创建了 TaskCaller 和 TaskExecutor 对象，并调用 executeTask 方法来执行任务。

作用

实现异步处理

在 Java 程序里，一些操作（像网络请求、文件读写等）会消耗大量时间。若采用同步方式，程序会暂停直到操作完成，这会严重影响性能和响应能力。通过回调机制，可在发起操作后继续执行其他任务，等操作完成时，再利用回调函数处理结果。

// 定义回调接口
interface DownloadCallback {
    void onDownloadComplete(String result);
}

// 模拟下载任务类
class Downloader {
    public void downloadFile(DownloadCallback callback) {
        new Thread(() -> {
            try {
                // 模拟下载耗时
                Thread.sleep(2000);
                String result = "文件下载完成";
                callback.onDownloadComplete(result);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
    }
}

// 调用者类
class Main {
    public static void main(String[] args) {
        Downloader downloader = new Downloader();
        downloader.downloadFile(result -> System.out.println(result));
        System.out.println("主线程继续执行其他任务");
    }
}

在上述代码中，Downloader 类的 downloadFile 方法开启一个新线程模拟文件下载，下载完成后通过回调函数 onDownloadComplete 通知调用者。主线程在发起下载请求后可继续执行其他任务。

事件驱动编程

在图形用户界面（GUI）编程、游戏开发等场景，经常需要处理各种事件（如按钮点击、鼠标移动）。回调机制能为每个事件绑定相应的回调函数，当事件触发时，执行对应的回调函数。

import javax.swing.*;
import java.awt.event.ActionEvent;
import java.awt.event.ActionListener;

public class ButtonExample {
    public static void main(String[] args) {
        JFrame frame = new JFrame("按钮示例");
        JButton button = new JButton("点击我");

        // 为按钮的点击事件绑定回调函数
        button.addActionListener(new ActionListener() {
            @Override
            public void actionPerformed(ActionEvent e) {
                System.out.println("按钮被点击了");
            }
        });

        frame.add(button);
        frame.setSize(300, 200);
        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        frame.setVisible(true);
    }
}

在这个例子中，通过 addActionListener 方法为按钮的点击事件绑定了一个回调函数 actionPerformed，当按钮被点击时，该回调函数会被调用。

代码解耦与复用

回调机制能够将不同的功能模块分离，降低代码间的耦合度，增强代码的复用性。一个模块可以定义通用的接口，其他模块通过实现这些接口来提供特定的行为。

// 定义通用的排序接口
interface SortingStrategy {
    int[] sort(int[] array);
}

// 具体的排序实现类
class BubbleSort implements SortingStrategy {
    @Override
    public int[] sort(int[] array) {
        // 冒泡排序实现
        int n = array.length;
        for (int i = 0; i < n - 1; i++) {
            for (int j = 0; j < n - i - 1; j++) {
                if (array[j] > array[j + 1]) {
                    int temp = array[j];
                    array[j] = array[j + 1];
                    array[j + 1] = temp;
                }
            }
        }
        return array;
    }
}

class QuickSort implements SortingStrategy {
    @Override
    public int[] sort(int[] array) {
        // 快速排序实现
        quickSort(array, 0, array.length - 1);
        return array;
    }

    private void quickSort(int[] array, int low, int high) {
        if (low < high) {
            int pi = partition(array, low, high);
            quickSort(array, low, pi - 1);
            quickSort(array, pi + 1, high);
        }
    }

    private int partition(int[] array, int low, int high) {
        int pivot = array[high];
        int i = (low - 1);
        for (int j = low; j < high; j++) {
            if (array[j] < pivot) {
                i++;
                int temp = array[i];
                array[i] = array[j];
                array[j] = temp;
            }
        }
        int temp = array[i + 1];
        array[i + 1] = array[high];
        array[high] = temp;
        return i + 1;
    }
}

// 排序工具类
class SortingUtils {
    public static int[] performSort(int[] array, SortingStrategy strategy) {
        return strategy.sort(array);
    }
}

// 调用示例
public class Main {
    public static void main(String[] args) {
        int[] array = {5, 4, 3, 2, 1};
        SortingStrategy bubbleSort = new BubbleSort();
        int[] sortedArray = SortingUtils.performSort(array, bubbleSort);
        for (int num : sortedArray) {
            System.out.print(num + " ");
        }
    }
}

在这个示例中，SortingUtils 类定义了一个通用的 performSort 方法，它接受一个 SortingStrategy 接口类型的参数。不同的排序算法（如冒泡排序、快速排序）可以通过实现 SortingStrategy 接口来提供不同的排序行为，这样 SortingUtils 类就可以复用，而不需要关心具体的排序实现。

回调实现异步详细介绍

示例代码

// 定义回调接口
interface TaskCallback {
    void onTaskCompleted(String result);
}

// 执行异步任务的类
class AsyncTaskExecutor {
    public void executeAsyncTask(TaskCallback callback) {
        // 创建并启动一个新线程来执行耗时任务
        new Thread(() -> {
            try {
                System.out.println("开始执行耗时任务...");
                // 模拟耗时操作，这里让线程休眠 2 秒
                Thread.sleep(2000);
                String result = "任务完成，这是任务结果";
                System.out.println("耗时任务执行完毕");
                // 任务完成后调用回调方法
                callback.onTaskCompleted(result);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }).start();
    }
}

public class AsyncCallbackWithoutCompletableFuture {
    public static void main(String[] args) {
        // 创建 AsyncTaskExecutor 实例，用于执行异步任务
        AsyncTaskExecutor executor = new AsyncTaskExecutor();
        // 定义回调逻辑
        TaskCallback callback = (result) -> {
            System.out.println("收到回调，结果是: " + result);
        };
        // 调用 executeAsyncTask 方法开始执行异步任务，并传入回调实例
        executor.executeAsyncTask(callback);
        System.out.println("主线程继续执行其他操作");
    }
}

代码执行过程

1. 程序启动，主线程开始执行 main 方法

当运行程序时，main 方法会被首先执行。此方法是整个程序的入口点。

2. 创建 AsyncTaskExecutor 实例

AsyncTaskExecutor executor = new AsyncTaskExecutor();

这行代码创建了 AsyncTaskExecutor 类的一个实例，该类负责执行异步任务。

3. 定义回调逻辑

TaskCallback callback = (result) -> {
    System.out.println("收到回调，结果是: " + result);
};

这里通过 Lambda 表达式实现了 TaskCallback 接口的 onTaskCompleted 方法，用于在任务完成后处理结果。

4. 调用 executeAsyncTask 方法

executor.executeAsyncTask(callback);
  • 调用 AsyncTaskExecutor 实例的 executeAsyncTask 方法，同时传入回调实例。
  • 在 executeAsyncTask 方法内部，会创建一个新的线程来执行耗时任务。

5. 新线程开始执行耗时任务

new Thread(() -> {
    try {
        System.out.println("开始执行耗时任务...");
        Thread.sleep(2000);
        String result = "任务完成，这是任务结果";
        System.out.println("耗时任务执行完毕");
        callback.onTaskCompleted(result);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
}).start();
  • 新线程启动后，会打印 “开始执行耗时任务…”。
  • 接着线程进入 2 秒的休眠状态，以此模拟耗时操作。

6. 主线程继续执行

System.out.println("主线程继续执行其他操作");

主线程不会等待新线程中的耗时任务完成，而是会继续执行后续代码，打印 “主线程继续执行其他操作”。这体现了异步处理的特性，即主线程不会被阻塞。

7. 新线程完成耗时任务

  • 2 秒休眠结束后，新线程会打印 “耗时任务执行完毕”。
  • 然后调用回调接口的 onTaskCompleted 方法，并将任务结果作为参数传入。

8. 触发回调

callback.onTaskCompleted(result);
  • 调用 callback 实例的 onTaskCompleted 方法，执行之前定义的回调逻辑。
  • 打印 “收到回调，结果是：任务完成，这是任务结果”。

import java.util.OptionalInt;
interface  Work {
    void work(String result);
}

class Boss implements Work{
    public void work(String result){
        if (result.equals("ok")){
            System.out.println("继续努力，给你加薪！！");
        }
        else {
            System.out.println("开除，开除！！");
        }
    }
}

class Manager implements Work {
    public void work(String result){
        if (result.equals("ok")){
            System.out.println("继续努力，给你吃个大饼！！");
        }
        else {
            System.out.println("废物一个！！");
        }
    }
}

class Employee {
    public Work accept;
    public void work () {
        System.out.println("疯狂努力敲代码！！！");
        accept.work("ok");
    }
    public void setAccept (Work accept){
        this.accept = accept;
    }
}

public class Main {
    @SuppressWarnings({"all"})
    public static void main(String[] args) {
        Employee employee = new Employee();
        Work texter = result -> {
            if (result.equals("ok")){
                System.out.println("虽然过了，但是，一般啊！！");
            }
            else {
                System.out.println("bug,bug,大bug！！");
            }
        };
        employee.setAccept(texter);
        employee.work();
        employee.setAccept((new Manager()));
        employee.work();

    }
}
