SQLite 数据库存储

SQL基础语法

1. 数据库操作

创建数据库

CREATE DATABASE database_name;

示例：

CREATE DATABASE testdb;

删除数据库

DROP DATABASE database_name;

示例：

DROP DATABASE testdb;

使用数据库

USE database_name;

示例：

USE testdb;

2. 表操作

创建表

CREATE TABLE table_name (
    column1 datatype,
    column2 datatype,
    column3 datatype,
    ...
);

示例：

CREATE TABLE customers (
    id INT PRIMARY KEY,
    name VARCHAR(255),
    age INT,
    email VARCHAR(255)
);

删除表

DROP TABLE table_name;

示例：

DROP TABLE customers;

修改表结构

  • 添加列
ALTER TABLE table_name
ADD column_name datatype;

示例：

ALTER TABLE customers
ADD address VARCHAR(255);
  • 删除列
ALTER TABLE table_name
DROP COLUMN column_name;

示例：

ALTER TABLE customers
DROP COLUMN address;

3. 数据操作

插入数据

INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);

示例：

INSERT INTO customers (id, name, age, email)
VALUES (1, 'John Doe', 30, 'john@example.com');

查询数据

SELECT column1, column2, ...
FROM table_name;

示例：

SELECT id, name, age
FROM customers;
  • 查询所有列
SELECT * FROM table_name;

示例：

SELECT * FROM customers;

更新数据

UPDATE table_name
SET column1 = value1, column2 = value2, ...
WHERE condition;

示例：

UPDATE customers
SET age = 31
WHERE id = 1;

删除数据

DELETE FROM table_name
WHERE condition;

示例：

DELETE FROM customers
WHERE id = 1;

SQLiteOpenHelper

添加

SQLiteOpenHelper:

package com.example.practice.database;

import android.content.ContentValues;
import android.content.Context;
import android.database.sqlite.SQLiteDatabase;
import android.database.sqlite.SQLiteOpenHelper;

import androidx.annotation.NonNull;
import androidx.annotation.Nullable;

public class UserDataHelper extends SQLiteOpenHelper {
    private static final String TABLE_NAME = "user_info";
    private static final String DB_NAME = "user.db";
    private static final int DB_VERSION = 3;
    private static UserDataHelper mHelper = null;
    private SQLiteDatabase mRDB = null;
    private SQLiteDatabase mWDB = null;

   private UserDataHelper(Context context){
       super(context,DB_NAME,null,DB_VERSION);
   }

   public static UserDataHelper getInstance(Context context){
       if (mHelper == null){
           mHelper = new UserDataHelper(context);
       }
       return mHelper;
   }

   public SQLiteDatabase openReadLink(){
       if (mRDB == null || !mRDB.isOpen()){
           mRDB = mHelper.getReadableDatabase();
       }
       return mRDB;
   }

    public SQLiteDatabase openWriteLink(){
        if (mWDB == null || !mWDB.isOpen()){
            mWDB = mHelper.getWritableDatabase();
        }
        return mWDB;
    }

    public void closeLink(){
       if(mRDB != null && mRDB.isOpen()){
           mRDB.close();
           mRDB = null;
       }
        if(mWDB != null && mWDB.isOpen()){
            mWDB.close();
            mWDB = null;
        }
    }

    @Override
    //构建一个包含name属性的表
    public void onCreate(@NonNull SQLiteDatabase db) {
        String sql = "CREATE TABLE " + TABLE_NAME + " (" +
                "id INTEGER PRIMARY KEY AUTOINCREMENT, " +
                "name TEXT);";
        db.execSQL(sql);
    }

    @Override
    public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {
        db.execSQL("DROP TABLE IF EXISTS " + TABLE_NAME);
        onCreate(db);
    }

    //添加数据
    public long insert(@NonNull User user){
        ContentValues values = new ContentValues();
        values.put("name",user.name);
        return mWDB.insert(TABLE_NAME,null,values);
    }
}

事务管理相关方法

  1. **beginTransaction()**：用于开始一个事务。调用该方法后，后续对数据库的操作都将作为事务的一部分，直到事务结束。
  2. **setTransactionSuccessful()**：设置事务的成功标志。在事务执行过程中，当所有操作都正确完成后，调用此方法标记事务成功。若未调用该方法，事务会在结束时自动回滚。
  3. **endTransaction()**：结束事务。执行该方法时，会检查事务是否已通过setTransactionSuccessful()设置为成功。如果已设置成功标志，则提交事务，将事务内的操作结果持久化到数据库；若未设置成功标志，则回滚事务，撤销事务内的所有操作。

使用示例

假设在一个简单的记账应用中，有两个账户（账户 A 和账户 B），要实现从账户 A 向账户 B 转账的操作，且这两个操作需作为一个事务来确保数据一致性。代码如下：

// 假设已获取SQLiteDatabase实例，名为db
db.beginTransaction(); // 开启事务
try {
    // 从账户A扣除转账金额，假设账户A的表名为account，id为1，金额字段为balance，转账金额为transferAmount
    db.execSQL("UPDATE account SET balance = balance -? WHERE id = 1", new String[]{String.valueOf(transferAmount)});
    // 向账户B增加转账金额，假设账户B的id为2
    db.execSQL("UPDATE account SET balance = balance +? WHERE id = 2", new String[]{String.valueOf(transferAmount)});
    db.setTransactionSuccessful(); // 设置事务成功标志
} catch (Exception e) {
    e.printStackTrace();
} finally {
    db.endTransaction(); // 结束事务，若设置了成功标志则提交，否则回滚
}

注意事项

  • 异常处理：在事务执行代码块中，要妥善处理可能出现的异常。一旦出现异常，通常不希望事务继续执行，避免数据不一致，如上述示例中在catch块捕获异常并打印堆栈信息。
  • 操作顺序：setTransactionSuccessful()方法需在事务执行的关键操作完成且都正确时调用，并且要在endTransaction()之前调用。
  • 自动提交模式：在 SQLite 中，默认情况下每个 SQL 语句都是一个自动提交的事务。但通过显式调用beginTransaction()等方法，可将多个 SQL 语句组合成一个事务进行处理 。
