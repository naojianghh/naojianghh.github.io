文件管理器

背景

操作系统具有对计算机硬件资源管理和调度的功能。文件是对占用了硬盘一定空间的对象的描述和抽象。考虑一般的文件具有文件名、大小和创建时间。文件管理在任何操作系统中都是必不可少的。文件管理器是用户用来观察和操作文件的一个软件。考虑一个简易的文件管理器，用户可以通过这个简易的文件管理器对某个目录下文件最大/小、文件名(字符串)字典序最大/最小的一些文件，即按XX排序功能，此外文件管理器具有删除和添加文件的功能。输入保证目录只有一级，操作随机。

输入及输出格式

第1行输入一个Q(1<=Q<=100000)，表示操作的次数。

第2-Q+1行输入一行操作序列

操作序列的格式为 op args

当op=1,即第一种操作时，args的格式为 folder name size 表示将要在某个folder目录(1<=folder<=5的整数)下添加文件名为name(1<=name<=100000的整数)的文件，文件大小为size(1<=size<=100000的整数)，如果文件名重复，就先删除原来的文件再进行1操作；

当op=2,即第二种操作时，args的格式为 folder name表示将要在某个folder目录(1<=folder<=5的整数)下删除文件名为name(1<=name<=100000的整数)的文件，如果文件名不存在，操作不执行；

当op=3,即第三种操作时，args的格式为 folder k 表示输出某个目录下文件大小最大至第k大的文件名，如果大小相同按文件名数字从小到大输出，如果不满k个文件只需要输出已有的文件名，如果文件夹为空或不存在则输出一个空行；

当op=4,即第四种操作时，args的格式为 folder k 表示输出某个目录下文件名数字大小最小至第k小的文件名，如果不满k个文件只需要输出已有的文件名，如果文件夹为空或不存在则输出一个空行；

保证1<= k <=10

备注

输出文件名以空格隔开，不输出多余的换行和空格

输入示例
10
2 3 74071
3 1 2
3 2 8
2 3 71337
3 3 7
3 2 7
4 2 10
4 3 6
4 3 4
3 1 4
输出示例

上方输出为8个空行

解题代码

#include<iostream>
#include<vector>

using namespace std;

class TreeNode {
public:
    int size;
    int name;
    TreeNode* left;
    TreeNode* right;
    TreeNode(int size,int name) : size(size),name(name), left(nullptr), right(nullptr){}

};

TreeNode* deleteNodebySize(TreeNode* root, int size);
TreeNode* deleteNodebyName(TreeNode* root, int name);

TreeNode* insertBSTbySize(TreeNode* root, int size,int name) {
    if (root == nullptr) {
        return new TreeNode(size,name);
    }
    if (size < root->size) {
        root->left = insertBSTbySize(root->left, size, name);
    }
    else if (size > root->size) {
        root->right = insertBSTbySize(root->right, size, name);
    }
    return root;
}

TreeNode* insertBSTbyName(TreeNode* root, int size, int name,TreeNode* root_Size) {
    if (root == nullptr) {
        return new TreeNode(size, name);
    }
    if (name < root->name) {
        root->left = insertBSTbyName(root->left, size, name, root_Size);
    }
    else if (name > root->name) {
        root->right = insertBSTbyName(root->right, size, name, root_Size);
    }
    else {
        root_Size = deleteNodebySize(root_Size,root->size);
        root_Size = insertBSTbySize(root_Size, size, name);
        root = deleteNodebyName(root, name);
        root = insertBSTbyName(root, size, name, root_Size);
    }
    return root;
}

TreeNode* findMin(TreeNode* root) {
    while (root->left != nullptr) {
        root = root->left;
    }
    return root;
}

TreeNode* deleteNodebyName(TreeNode* root, int name) {
    if (root == nullptr) return nullptr;

    if (name < root->name) {
        root->left = deleteNodebyName(root->left, name);
    }
    else if (name > root->name) {
        root->right = deleteNodebyName(root->right, name);
    }
    else {
        if (root->left == nullptr) {
            TreeNode* temp = root->right;
            delete root;
            return temp;
        }
        else if (root->right == nullptr) {
            TreeNode* temp = root->left;
            delete root;
            return temp;
        }
        else {
            TreeNode* temp = findMin(root->right);
            root->name = temp->name;
            root->right = deleteNodebyName(root->right, temp->name);
        }
    }
    return root;
}

TreeNode* deleteNodebySize(TreeNode* root, int size) {
    if (root == nullptr) return nullptr;

    if (size < root->size) {
        root->left = deleteNodebySize(root->left, size);
    }
    else if (size > root->size) {
        root->right = deleteNodebySize(root->right, size);
    }
    else {
        if (root->left == nullptr) {
            TreeNode* temp = root->right;
            delete root;
            return temp;
        }
        else if (root->right == nullptr) {
            TreeNode* temp = root->left;
            delete root;
            return temp;
        }
        else {
            TreeNode* temp = findMin(root->right);
            root->size = temp->size;
            root->right = deleteNodebySize(root->right, temp->size);
        }
    }
    return root;
}

TreeNode* searchBSTbyName(TreeNode* root, int name) {
    if (root == nullptr) {
        return root;
    }
    if (name < root->name) {
        return searchBSTbyName(root->left, name);
    }
    else if (name > root->name) {
        return searchBSTbyName(root->right, name);
    }
    else {
        return root;
    }
}

void inOrder(TreeNode* root,vector<int>& arr) {
    if (root == nullptr) return;
    inOrder(root->left,arr);   
    arr.push_back(root->name);
    inOrder(root->right,arr);
}

int main()
{
    int n;
    vector<TreeNode*> T_Name(6);
    vector<TreeNode*> T_Size(6);
    cin >> n;
    while (n--) {
        int op;
        cin >> op;
        if (op == 1) {
            int folder, name, size;
            cin >> folder >> name >> size;
            T_Name[folder] = insertBSTbyName(T_Name[folder], size, name, T_Size[folder]);
            T_Size[folder] = insertBSTbySize(T_Size[folder], size, name);
        }
        else if (op == 2) {
            int name,folder;
            cin >> folder >>  name;
            TreeNode* temp = searchBSTbyName(T_Name[folder], name);
            if (temp != nullptr) {
                int size = temp->size;
                T_Name[folder] = deleteNodebyName(T_Name[folder], name);
                T_Size[folder] = deleteNodebySize(T_Size[folder], size);
            }
        }
        else if (op == 3) {
            int k,folder;
            cin >> folder >> k;
            vector<int> arr;
            inOrder(T_Name[folder], arr);
            int i = arr.size() - 1;
            int count = arr.size() - k;
            while (i >= count && i >= 0) {
                cout << arr[i] << ' ';
                i--;
            }
            cout << endl;
        }
        else if (op == 4) {
            int k, folder;
            cin >> folder >> k;
            vector<int> arr;
            inOrder(T_Size[folder], arr);
            int i = 0;
            while (i < k && i < arr.size()) {
                cout << arr[i] << ' ';
                i++;
            }
        }
    }
    return 0;
}

核心思路

由于本题的操作次数和搜索量非常大，一般的线性数据存储和搜索时间会爆，因此采用二叉搜索树。

数据结构设计

  1. TreeNode 类：每个节点点存储两个关键信息
    • size：元素大小（用于按大小排序的 BST）
    • name：元素名称（用于按名称排序的 BST）
    • 左右指针（left/right）维护 BST 结构
  2. 双 BST 设计：
    • T_Name[folder]：按 name 升序的 BST（同一文件夹下的元素按名称组织）
    • T_Size[folder]：按 size 升序的 BST（同一文件夹下的元素按大小组织）
    • 两个 BST 存储相同的元素集合，只是排序依据不同，通过操作同步保证数据一致性

核心操作实现

1. 插入元素（op == 1）

  • 输入：文件夹编号 folder、元素名称 name、元素大小 size
  • 操作：
    • 调用 insertBSTbyName 向 T_Name[folder] 插入元素（按名称排序）
    • 调用 insertBSTbySize 向 T_Size[folder] 插入元素（按大小排序）
    • 若插入的 name 已存在，则先删除原元素（同步更新两个 BST），再插入新元素

2. 删除元素（op == 2）

  • 输入：文件夹编号 folder、元素名称 name
  • 操作：
    • 通过 searchBSTbyName 在 T_Name[folder] 中查找该名称对应的节点
    • 若找到，获取其size，同步执行：
      • 在 T_Name[folder] 中按 name 删除节点（deleteNodebyName）
      • 在 T_Size[folder] 中按 size 删除节点（deleteNodebySize）

3. 查询名称最大的 k 个元素（op == 3）

  • 输入：文件夹编号 folder、数量 k
  • 操作：
    • 对 T_Name[folder] 执行中序遍历（中序遍历 BST 可得到升序序列）
    • 取遍历结果的最后 k 个元素（即名称最大的 k 个），逆序输出

4. 查询大小最小的 k 个元素（op == 4）

  • 输入：文件夹编号 folder、数量 k
  • 操作：
    • 对 T_Size[folder] 执行中序遍历（得到按大小升序的序列）
    • 取遍历结果的前 k 个元素（即大小最小的 k 个）输出

BST 辅助函数

  • 插入：insertBSTbyName 和 insertBSTbySize 分别按 name 和 size 递归插入节点，维持 BST 性质
  • 删除：deleteNodebyName 和 deleteNodebySize 处理三种节点情况（叶子、单孩子、双孩子），双孩子节点用右子树最小节点替换
  • 查找：searchBSTbyName 按名称递归查找节点
  • 中序遍历：inOrder 递归遍历 BST，收集节点信息（用于查询操作）
