相机和相册的对接

2025/9/6 Android

复刻bilibiliUI项目学习
Github项目链接： naojianghh/bilibili

照相和相册的 API 对接实现详解

在仿哔哩哔哩项目中，用户可以通过相机拍摄或从相册选择图片来设置收藏夹封面，这一功能主要通过 Android 系统提供的相机和相册 API 实现，核心代码集中在CreateFolderActivity中。以下是具体实现细节：

一、权限配置

使用相机和相册功能需要在AndroidManifest.xml中声明相关权限：

<!-- 相机权限 -->
<uses-permission android:name="android.permission.CAMERA" />
<!-- 存储权限（针对Android 10以下） -->
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />
<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />
<!-- 声明明显式使用相机硬件 -->
<uses-feature
    android:name="android.hardware.camera"
    android:required="true" />

同时在application标签中添加文件提供者配置，用于安全地共享相机拍摄的图片：

<application ...>
    <!-- 文件提供者配置，用于相机照片的安全访问 -->
    <provider
        android:name="androidx.core.content.FileProvider"
        android:authorities="com.naojianghh.bilibili3"
        android:exported="false"
        android:grantUriPermissions="true">
        <meta-data
            android:name="android.support.FILE_PROVIDER_PATHS"
            android:resource="@xml/file_paths" />
    </provider>
</application>

三、相机 API 对接

1. 权限检查

在打开相机前检查并请求必要权限：

private void checkPermissionsAndOpenCamera() {
    // 检查相机和存储权限
    if (ContextCompat.checkSelfPermission(this, Manifest.permission.CAMERA)
            != PackageManager.PERMISSION_GRANTED ||
        ContextCompat.checkSelfPermission(this, Manifest.permission.WRITE_EXTERNAL_STORAGE)
            != PackageManager.PERMISSION_GRANTED) {

        // 请求权限
        ActivityCompat.requestPermissions(this,
                new String[]{Manifest.permission.CAMERA,
                            Manifest.permission.WRITE_EXTERNAL_STORAGE},
                REQUEST_PERMISSIONS);
    } else {
        // 权限已授予，直接打开相机
        openCamera();
    }
}

2. 打开相机

使用MediaStore.ACTION_IMAGE_CAPTURE意图打开系统相机：

private void openCamera() {
    Intent takePictureIntent = new Intent(MediaStore.ACTION_IMAGE_CAPTURE);
    // 检查是否有相机应用可以响应
    if (takePictureIntent.resolveActivity(getPackageManager()) != null) {
        File photoFile = null;
        try {
            // 创建临时图片文件
            photoFile = createImageFile();
        } catch (IOException ex) {
            ex.printStackTrace();
        }
        if (photoFile != null) {
            // 通过FileProvider获取安全的Uri
            Uri photoURI = FileProvider.getUriForFile(this,
                    "com.naojianghh.bilibili3", // 与Manifest中配置的authorities一致
                    photoFile);
            // 设置图片保存路径
            takePictureIntent.putExtra(MediaStore.EXTRA_OUTPUT, photoURI);
            // 启动相机，等待返回结果
            startActivityForResult(takePictureIntent, REQUEST_CAMERA);
        }
    }
}

3. 创建图片文件

private File createImageFile() throws IOException {
    // 生成唯一文件名（基于时间戳）
    String timeStamp = new SimpleDateFormat("yyyyMMdd_HHmmss", Locale.getDefault()).format(new Date());
    String imageFileName = "JPEG_" + timeStamp + "_";
    // 获取应用私有图片目录
    File storageDir = getExternalFilesDir(Environment.DIRECTORY_PICTURES);
    // 创建临时文件
    File image = File.createTempFile(
            imageFileName,  // 文件名前缀
            ".jpg",         // 文件后缀
            storageDir      // 存储目录
    );
    // 保存图片路径用于后续处理
    currentPhotoPath = image.getAbsolutePath();
    return image;
}

四、相册 API 对接

使用ACTION_OPEN_DOCUMENT意图打开系统相册，支持持久化访问权限：

private void openGallery() {
    Intent intent = new Intent(Intent.ACTION_OPEN_DOCUMENT);
    intent.setType("image/*"); // 只显示图片类型
    // 请求持久化访问权限
    intent.addFlags(Intent.FLAG_GRANT_PERSISTABLE_URI_PERMISSION);
    intent.addFlags(Intent.FLAG_GRANT_READ_URI_PERMISSION);
    // 启动相册，等待返回结果
    startActivityForResult(intent, REQUEST_GALLERY);
}

五、处理返回结果

在onActivityResult中处理相机和相册返回的图片：

@Override
protected void onActivityResult(int requestCode, int resultCode, @Nullable Intent data) {
    super.onActivityResult(requestCode, resultCode, data);
    
    // 处理相机返回结果
    if (requestCode == REQUEST_CAMERA && resultCode == RESULT_OK) {
        // 通过路径加载图片
        Bitmap bitmap = BitmapFactory.decodeFile(currentPhotoPath);
        ivSelectedImage.setImageBitmap(bitmap);
        // 保存图片路径用于后续存储
        selectedImageIdentifier = currentPhotoPath;
    }
    // 处理相册返回结果
    else if (requestCode == REQUEST_GALLERY && resultCode == RESULT_OK && data != null) {
        Uri selectedImageUri = data.getData();
        if (selectedImageUri != null) {
            // 显示选中的图片
            ivSelectedImage.setImageURI(selectedImageUri);
            
            try {
                // 获取持久化访问权限
                int takeFlags = data.getFlags() & (Intent.FLAG_GRANT_READ_URI_PERMISSION | 
                                                  Intent.FLAG_GRANT_WRITE_URI_PERMISSION);
                getContentResolver().takePersistableUriPermission(selectedImageUri, takeFlags);
                // 保存带权限标记的Uri
                selectedImageIdentifier = selectedImageUri.toString() + "|with_permission";
            } catch (SecurityException e) {
                Log.e("PermissionError", "无法获取持久化权限: " + e.getMessage());
                selectedImageIdentifier = selectedImageUri.toString();
                Toast.makeText(this, "无法永久访问该图片，仅本次有效", Toast.LENGTH_SHORT).show();
            }
        }
    }
}

六、图片展示与存储

在FolderActivity中加载并展示选中的图片（区分相机图片和相册图片）：

// 获取数据库中存储的封面路径
String coverPath = databaseHelper.getFolderCover(folderName);
if (coverPath != null && !coverPath.isEmpty()) {
    setPicture.setVisibility(View.VISIBLE);
    // 区分封面类型：路径（相机图）/ Uri字符串（相册图）
    if (coverPath.startsWith("/")) {
        // 相机图：通过路径加载图片（压缩避免OOM）
        Bitmap compressedBitmap = decodeCompressedBitmap(coverPath, 80, 80);
        setPicture.setImageBitmap(compressedBitmap);
    } else if (coverPath.startsWith("content://")) {
        // 相册图：通过Uri加载
        String[] coverParts = coverPath.split("\\|");
        Uri coverUri = Uri.parse(coverParts[0]);
        
        try {
            setPicture.setImageURI(coverUri);
        } catch (SecurityException e) {
            // 权限不足时的备选方案
            try (InputStream inputStream = getContentResolver().openInputStream(coverUri)) {
                Bitmap bitmap = BitmapFactory.decodeStream(inputStream);
                setPicture.setImageBitmap(bitmap);
            } catch (Exception ex) {
                ex.printStackTrace();
                setPicture.setVisibility(View.GONE);
            }
        }
    }
} else {
    setPicture.setVisibility(View.GONE);
}

七、关键技术点

  1. 权限管理：
    • 动态请求相机和存储权限
    • 使用takePersistableUriPermission获取相册图片的持久化访问权
  2. 安全机制：
    • 通过FileProvider生成 content:// 类型的 Uri，避免暴露文件路径
    • 对相机拍摄的图片进行压缩处理，防止 OOM
  3. 兼容性处理：
    • 区分相机图片（文件路径）和相册图片（Uri）的不同处理方式
    • 处理权限获取失败的异常情况
  4. 数据存储：
    • 相机图片存储在应用私有目录，避免权限问题
    • 相册图片存储 Uri 字符串，配合权限标记实现持久访问
