Android异步任务与多线程详解

2025/9/10 Android

在 Android 开发中，多线程是处理耗时操作（如网络请求、文件读写、复杂计算等）的核心技术，能有效避免主线程（UI 线程）阻塞，保证应用的流畅性。

异步任务

一、核心概念

1. 同步任务（Synchronous）

  • 执行方式：任务按顺序执行，前一个任务完成后，才会开始下一个任务。
  • 阻塞特性：当前线程会被阻塞，等待任务完成后再继续执行后续代码。
  • 形象比喻：类似打电话，必须等对方说完，你才能开始说，中间不能做其他事。

2. 异步任务（Asynchronous）

  • 执行方式：任务启动后，不等待其完成，直接执行后续代码，任务在后台独立运行。
  • 非阻塞特性：当前线程不会被阻塞，可以继续处理其他任务。
  • 结果处理：任务完成后，通常通过回调（Callback）、事件通知等方式返回结果。
  • 形象比喻：类似发邮件，发送后不必等待对方回复，可以继续做其他事，对方回复时会收到通知。

二、代码示例对比

1. 同步任务示例（Android 中主线程执行耗时操作的错误示范）

// 主线程中执行同步任务（错误示例！会导致ANR）
fun syncTask() {
    println("任务1开始")
    // 模拟耗时操作（如网络请求）
    Thread.sleep(3000) // 主线程被阻塞3秒
    println("任务1完成")
    
    println("任务2开始") // 必须等任务1完成才执行
    Thread.sleep(2000)
    println("任务2完成")
}

问题：主线程被阻塞，界面会冻结，超过 5 秒触发 ANR。

2. 异步任务示例（使用 Kotlin 协程）

// 异步执行任务（推荐方式）
fun asyncTask() {
    println("主线程继续执行其他操作")
    
    // 启动异步任务（在IO线程执行）
    lifecycleScope.launch(Dispatchers.IO) {
        println("异步任务1开始")
        delay(3000) // 模拟耗时，不阻塞主线程
        println("异步任务1完成")
        
        // 切换到主线程更新UI
        withContext(Dispatchers.Main) {
            textView.text = "任务1完成"
        }
    }
    
    // 无需等待任务1，直接执行后续代码
    println("任务2开始（主线程）")
    // 主线程可以继续处理用户交互等操作
}

特点：异步任务在后台执行，主线程不阻塞，界面保持流畅。

多线程

一、Android 线程基础

  1. 主线程（UI 线程）
    • 负责处理 UI 渲染和用户交互，不能执行耗时操作（超过 5 秒会触发 ANR - Application Not Responding）。
    • 所有 UI 操作必须在主线程执行，子线程不能直接更新 UI。
  2. 子线程（工作线程）
    • 用于执行耗时操作，避免阻塞主线程。
    • 不能直接操作 UI，需通过特定机制通知主线程更新 UI。

二、多线程实现方式

1. Thread + Handler（传统方式）

通过Thread执行耗时操作，Handler实现线程间通信（子线程通知主线程更新 UI）。

示例代码：

// 主线程中定义Handler（用于接收子线程消息）
private val mHandler = object : Handler(Looper.getMainLooper()) {
    override fun handleMessage(msg: Message) {
        super.handleMessage(msg)
        // 在主线程更新UI
        when (msg.what) {
            1 -> textView.text = "耗时操作完成"
        }
    }
}

// 启动子线程执行耗时操作
fun startTask() {
    Thread {
        // 模拟耗时操作（如网络请求）
        Thread.sleep(3000)
        
        // 发送消息到主线程
        val msg = Message.obtain()
        msg.what = 1 // 消息标识
        mHandler.sendMessage(msg)
    }.start()
}

2. AsyncTask（已过时，了解即可）

封装了Thread和Handler，简化异步操作，但在 Android 11 中已被标记为过时，推荐使用Coroutine或Executor替代。

示例代码：

// 泛型参数：Params（输入参数）、Progress（进度）、Result（结果）
class MyAsyncTask : AsyncTask<Void, Int, String>() {
    // 子线程中执行耗时操作
    override fun doInBackground(vararg params: Void?): String {
        for (i in 0..100) {
            Thread.sleep(50)
            publishProgress(i) // 发布进度
        }
        return "任务完成"
    }

    // 主线程中更新进度（在publishProgress后调用）
    override fun onProgressUpdate(vararg values: Int?) {
        super.onProgressUpdate(*values)
        progressBar.progress = values[0] ?: 0
    }

    // 主线程中处理结果（任务完成后调用）
    override fun onPostExecute(result: String?) {
        super.onPostExecute(result)
        textView.text = result
    }
}

// 启动任务
MyAsyncTask().execute()

3. ThreadPoolExecutor（线程池）

通过线程池管理线程，避免频繁创建 / 销毁线程的开销，提高性能。Android 推荐使用Executors工具类创建线程池。

示例代码：

// 创建线程池（核心线程数3，最大线程数5，空闲时间10秒）
private val executor = ThreadPoolExecutor(
    3, 5, 10, TimeUnit.SECONDS,
    LinkedBlockingQueue(),
    Executors.defaultThreadFactory(),
    ThreadPoolExecutor.AbortPolicy() // 拒绝策略
)

// 执行任务
fun executeTask() {
    executor.execute {
        // 子线程执行耗时操作
        Thread.sleep(3000)
        
        // 切换到主线程更新UI
        runOnUiThread {
            textView.text = "线程池任务完成"
        }
    }
}

// 页面销毁时关闭线程池
override fun onDestroy() {
    super.onDestroy()
    executor.shutdown()
}

4. Kotlin Coroutine（推荐）

Kotlin 协程是轻量级线程，语法简洁，通过Dispatchers指定线程类型，轻松实现线程切换。

使用步骤：

  1. 添加依赖（AndroidX 项目默认包含）：
dependencies {
    implementation 'org.jetbrains.kotlinx:kotlinx-coroutines-android:1.6.4'
}
  1. 示例代码：
// 启动协程（ lifecycleScope 是AndroidX提供的生命周期感知协程作用域）
fun startCoroutineTask() {
    lifecycleScope.launch {
        // 切换到IO线程执行耗时操作
        val result = withContext(Dispatchers.IO) {
            // 模拟网络请求
            Thread.sleep(3000)
            "协程任务结果"
        }
        
        // 自动切换回主线程更新UI
        textView.text = result
    }
}

核心优势：

  • 语法简洁，避免回调嵌套（“回调地狱”）。
  • 自动绑定生命周期（lifecycleScope），防止内存泄漏。
  • 通过Dispatchers灵活切换线程（Main主线程、IO输入输出、Default计算密集型）。

三、线程通信核心机制

  1. Handler + Looper + MessageQueue

    • MessageQueue：消息队列，存储子线程发送的消息。
    • Looper：循环读取MessageQueue中的消息，分发给对应Handler。
    • Handler：发送和处理消息，实现线程间通信。

  2. runOnUiThread
    Activity 提供的方法，简化子线程切换到主线程更新 UI：

    Thread {
    // 耗时操作
    runOnUiThread {
        textView.text = "更新UI"
    }
}.start()

  3. View.post()
    视图提供的方法，将任务 post 到主线程执行：

    textView.post {
    textView.text = "主线程更新"
}

四、注意事项

  1. 避免内存泄漏
    • 子线程持有 Activity 引用时，需在页面销毁时停止线程。
    • 使用lifecycleScope（协程）或WeakReference（弱引用）管理生命周期。
  2. UI 操作限制
    • 所有 UI 更新必须在主线程执行，子线程直接操作会抛出异常。
  3. 线程池管理
    • 避免创建大量线程，优先使用线程池复用线程。
    • 不再使用时需调用shutdown()关闭线程池。

总结

  • 简单场景：使用Thread + Handler或View.post()。
  • 复杂异步任务：推荐Kotlin Coroutine（简洁、安全）。
  • 多任务并发：使用ThreadPoolExecutor管理线程资源。
