java与kotlin基础语法区别

2025/5/13 Android

Java 和 Kotlin 作为 JVM 平台上的主流语言，在基础语法上有诸多差异，Kotlin 更注重简洁性、安全性和现代编程范式。

1. 变量声明

  • Java：需显式指定类型，区分声明与初始化，变量默认可变（需用final声明不可变）：

    String name; // 仅声明
final int age = 20; // 不可变变量（必须初始化）
int score = 90; // 可变变量

  • Kotlin：类型可推断，用val（不可变）和var（可变）声明，必须初始化（或延迟初始化）：

    val name: String = "Alice" // 不可变，类型可省略（推断为String）
var age = 20 // 可变，类型自动推断为Int
lateinit var address: String // 延迟初始化（非基本类型）

2. 函数定义

  • Java：必须用public/private等修饰符，返回类型在前，参数需显式类型：

    public int add(int a, int b) {
    return a + b;
}

  • Kotlin：用fun关键字，返回类型在后（可省略Unit，类似void），参数类型可推断但通常显式声明：

    fun add(a: Int, b: Int): Int { // 返回类型Int
    return a + b
}

fun printHello() { // 无返回值（默认Unit）
    println("Hello")
}

// 单表达式函数（简化写法）
fun multiply(a: Int, b: Int) = a * b

3. 空安全

  • Java：默认允许 null，需手动判空，易出现NullPointerException：

    String str = null;
int length = str.length(); // 运行时异常

  • Kotlin：默认禁止 null，通过?标记可空类型，强制判空：

    var nonNullStr: String = "abc"
nonNullStr = null // 编译错误（不可空）

var nullableStr: String? = null
// 安全调用（返回null，不崩溃）
val length = nullableStr?.length 
// 非空断言（确认不为null，否则崩溃）
val len = nullableStr!!.length 
// 空值默认值
val l = nullableStr?.length ?: 0

4. 条件语句

  • Java：if-else是语句，无返回值；switch需break，支持常量：

    int x = 10;
String result;
if (x > 5) {
    result = "Big";
} else {
    result = "Small";
}

switch (x) {
    case 10:
        System.out.println("Ten");
        break;
    default:
        System.out.println("Other");
}

  • Kotlin：if是表达式（有返回值）；when替代switch，更灵活：

    val x = 10
val result = if (x > 5) "Big" else "Small" // 表达式直接赋值

when (x) {
    10 -> println("Ten")
    in 1..9 -> println("Single digit") // 支持范围
    else -> println("Other")
}

// when作为表达式返回值
val desc = when {
    x > 100 -> "Huge"
    x < 0 -> "Negative"
    else -> "Normal"
}

5. 循环语句

  • Java：for循环需下标或迭代器，while与do-while：

    List<String> list = Arrays.asList("a", "b");
for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}
for (String s : list) { // 增强for
    System.out.println(s);
}

  • Kotlin：支持for-in循环，范围运算符..，更简洁：

    val list = listOf("a", "b")
for (s in list) {
    println(s)
}

// 范围循环
for (i in 1..5) { // 1到5（含）
    println(i)
}
for (i in 5 downTo 1 step 2) { // 5,3,1
    println(i)
}

6. 类与构造函数

  • Java：类默认不可继承（需extends），构造函数与类名相同：

    public class Person {
    private String name;
    private int age;
    
    // 构造函数
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    // getter/setter（必须手动定义）
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
}

  • Kotlin：类默认final（需open才能继承），主构造函数直接声明在类头，自动生成getter/setter：

    // 数据类（自动生成equals、hashCode等）
data class Person(
    val name: String, // val：只读（仅getter）
    var age: Int      // var：可读写（getter+setter）
)

// 主构造函数带逻辑（init代码块）
class Student(val id: Int) {
    init {
        println("Student id: $id")
    }
}

7. 继承与接口

  • Java：用extends继承类，implements实现接口，单继承多实现：

    public class Student extends Person implements Study {
    // 实现父类和接口方法
}

  • Kotlin：用:代替extends和implements，父类构造函数需显式调用：

    open class Person // 需open才能被继承
interface Study

class Student : Person(), Study { // 继承类+实现接口
    // 实现方法
}

8. 其他语法糖

  • 字符串模板：Kotlin 用$直接嵌入变量，Java 需+拼接：

    val name = "Bob"
println("Hello, $name") // 输出 Hello, Bob

  • 集合操作：Kotlin 提供更简洁的集合 API（如listOf、mapOf），支持链式操作：

    val numbers = listOf(1, 2, 3)
val evenNumbers = numbers.filter { it % 2 == 0 } // 过滤偶数

  • 扩展函数：Kotlin 可给现有类添加方法，无需继承：

    fun String.addSuffix(): String {
    return this + "_suffix"
}
"test".addSuffix() // 结果为 "test_suffix"

总结

Kotlin 在语法上简化了 Java 的冗余代码（如省略分号、自动生成 getter/setter），强化了空安全和函数式编程特性，同时保持与 Java 的完全互操作性。对于熟悉 Java 的开发者，Kotlin 学习成本较低，且能显著提升开发效率。
