左式堆

2025/11/15 算法

左式堆是一种可合并堆（Mergeable Heap），核心优势是支持高效的「合并操作」（时间复杂度 O (logN)），同时保留堆的「极值优先」特性（大根堆 / 小根堆）。它通过引入「零路径长度（NPL）」约束左子树的深度，确保合并时仅需沿 “左偏路径” 遍历，避免退化。

左式堆详解

左式堆（Leftist Heap）是一种可合并堆（Mergeable Heap），核心优势是支持高效的合并操作（时间复杂度 O (log n)），同时保持堆的性质（大根堆 / 小根堆）。它通过 “左偏性”（Leftist Property）保证合并效率，广泛应用于需要频繁合并堆的场景（如 K 路归并、任务调度等）。

一、核心概念

1. 左式堆的定义

左式堆是一棵二叉树，满足两个条件：

  1. 堆序性：任意节点的值 ≥ 其子节点的值（大根堆），或 ≤ 其子节点的值（小根堆）（本文以小根堆为例）；
  2. 左偏性：任意节点的零路径长度（Null Path Length, NPL） 满足：左子树的 NPL ≥ 右子树的 NPL。

2. 零路径长度（NPL）

  • 定义：节点到最近的 “空节点” 的路径长度（空节点的 NPL 为 - 1，叶子节点的 NPL 为 0）；
  • 公式：NPL(node) = 1 + min(NPL(node->left), NPL(node->right))；
  • 作用：左偏性保证了树的 “左重右轻”，合并时只需优先处理右子树，避免最坏情况。

3. 左式堆的特点

  • 不是完全二叉树，无需数组存储，采用链式结构；
  • 合并是核心操作，插入、删除堆顶等操作均可基于合并实现；
  • 平均 / 最坏时间复杂度：合并、插入、删除堆顶均为 O (log n)。

二、数据结构设计

1. 节点结构

每个节点需存储：值、左子指针、右子指针、NPL 值。

template <typename T>
struct LeftistNode {
    T val;                  // 节点值
    LeftistNode<T>* left;   // 左子节点指针
    LeftistNode<T>* right;  // 右子节点指针
    int npl;                // 零路径长度

    // 构造函数
    LeftistNode(T value) : val(value), left(nullptr), right(nullptr), npl(0) {}
};

2. 左式堆类

封装节点指针、堆操作（合并、插入、删除堆顶、查询堆顶等）：

template <typename T>
class LeftistHeap {
private:
    LeftistNode<T>* root;  // 堆的根节点

    // 辅助函数：合并两个左式堆（核心）
    LeftistNode<T>* merge(LeftistNode<T>* a, LeftistNode<T>* b);
    // 辅助函数：更新节点的NPL值
    void updateNPL(LeftistNode<T>* node);
    // 辅助函数：释放堆内存（递归）
    void destroy(LeftistNode<T>* node);

public:
    LeftistHeap() : root(nullptr) {}  // 构造空堆
    ~LeftistHeap() { destroy(root); } // 析构函数（释放内存）

    bool isEmpty() const;       // 判断堆是否为空
    T getMin() const;           // 获取堆顶（小根堆的最小值）
    void insert(T value);       // 插入元素
    void deleteMin();           // 删除堆顶元素
    void merge(LeftistHeap<T>& other);  // 合并另一个左式堆
};

三、核心操作实现

1. 合并操作（最关键）

合并两个左式堆的核心思路：

  1. 递归选择较小根节点的堆，将另一堆合并到其右子树（保证堆序性）；
  2. 合并后检查左偏性，若左子树 NPL < 右子树 NPL，则交换左右子树；
  3. 更新当前节点的 NPL 值（基于左右子树的最小 NPL 计算）。

实现代码：

template <typename T>
LeftistNode<T>* LeftistHeap<T>::merge(LeftistNode<T>* a, LeftistNode<T>* b) {
    // 边界条件：一个堆为空，直接返回另一个堆
    if (a == nullptr) return b;
    if (b == nullptr) return a;

    // 保证a的根节点更小（小根堆），否则交换a和b
    if (a->val > b->val) {
        swap(a, b);
    }

    // 递归合并a的右子树和b
    a->right = merge(a->right, b);

    // 检查左偏性：若右子树NPL更大，交换左右子树
    if (a->left == nullptr || a->left->npl < a->right->npl) {
        swap(a->left, a->right);
    }

    // 更新当前节点的NPL
    updateNPL(a);

    return a;
}

// 更新节点的NPL：1 + min(左子NPL, 右子NPL)
template <typename T>
void LeftistHeap<T>::updateNPL(LeftistNode<T>* node) {
    if (node == nullptr) return;
    // 空节点NPL为-1，叶子节点的左右子均为空，NPL=0
    if (node->left == nullptr && node->right == nullptr) {
        node->npl = 0;
    } else if (node->right == nullptr) {
        // 右子为空，min为右子的-1，故NPL=0
        node->npl = 0;
    } else {
        node->npl = 1 + min(node->left->npl, node->right->npl);
    }
}

2. 插入操作

插入本质是合并 “单个节点的堆” 和原堆，直接复用 merge 函数：

template <typename T>
void LeftistHeap<T>::insert(T value) {
    LeftistNode<T>* newNode = new LeftistNode<T>(value);
    root = merge(root, newNode);  // 合并原堆和新节点堆
}

3. 删除堆顶操作

删除堆顶（最小值）的思路：

  1. 堆顶的左右子树构成两个左式堆；
  2. 合并这两个子堆，作为新的堆根；
  3. 释放原堆顶节点的内存。

实现代码：

template <typename T>
void LeftistHeap<T>::deleteMin() {
    if (isEmpty()) {
        throw runtime_error("Heap is empty, cannot delete min!");
    }
    LeftistNode<T>* oldRoot = root;
    // 合并左右子堆，作为新根
    root = merge(root->left, root->right);
    delete oldRoot;  // 释放原堆顶内存
}

4. 其他辅助操作

// 判断堆是否为空
template <typename T>
bool LeftistHeap<T>::isEmpty() const {
    return root == nullptr;
}

// 获取堆顶（最小值）
template <typename T>
T LeftistHeap<T>::getMin() const {
    if (isEmpty()) {
        throw runtime_error("Heap is empty, no min value!");
    }
    return root->val;
}

// 合并另一个左式堆（外部接口）
template <typename T>
void LeftistHeap<T>::merge(LeftistHeap<T>& other) {
    root = merge(this->root, other.root);
    other.root = nullptr;  // 避免对方堆重复释放内存
}

// 递归释放堆内存
template <typename T>
void LeftistHeap<T>::destroy(LeftistNode<T>* node) {
    if (node == nullptr) return;
    destroy(node->left);
    destroy(node->right);
    delete node;
}

四、测试代码

验证左式堆的核心功能（插入、删除堆顶、合并）：

#include <iostream>
#include <vector>
using namespace std;

int main() {
    // 测试1：插入和删除堆顶
    LeftistHeap<int> heap1;
    heap1.insert(5);
    heap1.insert(3);
    heap1.insert(7);
    heap1.insert(1);
    cout << "Heap1 min: " << heap1.getMin() << endl;  // 输出1
    heap1.deleteMin();
    cout << "After delete min, Heap1 min: " << heap1.getMin() << endl;  // 输出3

    // 测试2：合并两个堆
    LeftistHeap<int> heap2;
    heap2.insert(2);
    heap2.insert(4);
    heap1.merge(heap2);
    cout << "After merge, Heap1 min: " << heap1.getMin() << endl;  // 输出2
    heap1.deleteMin();
    cout << "After delete min, Heap1 min: " << heap1.getMin() << endl;  // 输出3

    return 0;
}

输出结果：

Heap1 min: 1
After delete min, Heap1 min: 3
After merge, Heap1 min: 2
After delete min, Heap1 min: 3

五、时间复杂度分析

操作 时间复杂度 说明
合并（merge） O(log n) 每次递归深入一层，树高为 O (log n)
插入（insert） O(log n) 等价于合并单个节点，复用 merge
删除堆顶 O(log n) 合并两个子堆，复用 merge
查询堆顶 O(1) 直接访问根节点

注：左式堆的树高最坏为 O (log n)（因左偏性限制了右子树深度），故所有操作均为对数时间。
