CSS3弹性盒子模型

2025/11/10 web CSS

CSS3 弹性盒子模型（Flexible Box，简称 Flexbox）是一种一维布局模型，专为解决传统布局（float、position）的对齐复杂、自适应繁琐等问题设计，现已成为前端布局的首选方案。它通过「容器 - 项目」的结构，轻松实现元素的对齐、分布、伸缩和自适应，兼容性覆盖所有现代浏览器（IE11 部分支持）。

CSS3 弹性盒子（Flexbox

一、核心原理与概念

Flexbox 的布局逻辑围绕「主轴」和「交叉轴」展开，核心是通过控制容器和项目的属性，实现灵活的布局效果。先明确以下关键概念：

概念 定义与说明
弹性容器（Flex Container） 父元素通过 display: flex 或 display: inline-flex 声明，其直接子元素自动成为弹性项目
弹性项目（Flex Item） 弹性容器的直接子元素（不包含嵌套子元素），默认沿主轴排列
主轴（Main Axis） 项目排列的主方向，由 flex-direction 控制（默认水平向左→右）
交叉轴（Cross Axis） 与主轴垂直的方向（默认垂直向上→下），所有交叉轴相关属性（如 align-items）均作用于此方向
主轴起点 / 终点（Main Start/End） 主轴的开始和结束位置（如水平主轴的左 / 右）
交叉轴起点 / 终点（Cross Start/End） 交叉轴的开始和结束位置（如垂直交叉轴的上 / 下）
基线（Baseline） 项目内文字的基准线（align-items: baseline 时按此对齐）

二、基础入门：创建弹性容器

1. 声明弹性容器

给父元素添加 display 属性，即可将其变为弹性容器，两种取值区别如下：

/* 1. 块级弹性容器：独占一行，宽度默认 100%（常用） */
.container {
  display: flex;
}

/* 2. 行内弹性容器：宽度自适应内容，与其他行内元素同行排列 */
.container {
  display: inline-flex;
}

2. 项目的默认行为

容器声明为 flex 后，子项目（Flex Item）会自动呈现以下特性：

  • 沿主轴（默认水平）排列，不再独占一行（打破块级元素默认行为）
  • 项目高度默认拉伸至与容器等高（align-items: stretch）
  • 项目不会换行，空间不足时自动收缩（flex-shrink: 1）
  • 忽略 float、clear、vertical-align 属性（失效）

三、容器属性（控制整体布局）

弹性容器的属性用于定义项目的排列方向、换行规则、对齐方式等，共 6 个核心属性，是 Flexbox 布局的核心。

1. flex-direction：控制主轴方向（项目排列方向）

决定主轴的方向，直接改变项目的排列顺序（水平 / 垂直），取值如下：

取值 主轴方向 项目排列效果 适用场景
row（默认） 水平（左→右） 从左到右横向排列 导航栏、水平布局
row-reverse 水平（右→左） 从右到左横向排列 反向水平布局
column 垂直（上→下） 从上到下纵向排列 垂直导航、表单布局
column-reverse 垂直（下→上） 从下到上纵向排列 反向垂直布局

示例：垂直排列项目

.container {
  display: flex;
  flex-direction: column; /* 主轴垂直，项目从上到下排列 */
  height: 200px; /* 容器高度决定交叉轴空间 */
}

2. flex-wrap：控制项目是否换行

默认情况下，项目会压缩在一行（nowrap），通过 flex-wrap 控制溢出时的换行规则：

取值 效果说明 适用场景
nowrap（默认） 不换行，项目自动收缩以适应容器宽度 少量项目的水平布局
wrap 换行，溢出的项目在下一行排列（从上到下） 响应式卡片、多项目布局
wrap-reverse 反向换行，溢出的项目在上一行排列（从下到上） 特殊反向布局

示例：自动换行布局

.container {
  display: flex;
  flex-wrap: wrap; /* 空间不足时自动换行 */
  width: 500px; /* 容器固定宽度，触发换行 */
}

3. flex-flow：flex-direction + flex-wrap 简写

为简化代码，将两个属性合并为 flex-flow，默认值 row nowrap：

/* 水平排列 + 自动换行（常用组合） */
.container {
  display: flex;
  flex-flow: row wrap;
}

/* 垂直排列 + 不换行 */
.container {
  flex-flow: column nowrap;
}

4. justify-content：主轴对齐方式（核心）

控制项目在主轴方向的对齐和分布，是最常用的属性之一（以 flex-direction: row 为例，主轴为水平方向）：

取值 效果说明 视觉示意（水平主轴） 适用场景
flex-start（默认） 靠主轴起点对齐（左对齐） [项目 1 项目 2 项目 3] —— 左对齐布局
flex-end 靠主轴终点对齐（右对齐） —— [项目 1 项目 2 项目 3] 右对齐布局
center 主轴居中对齐 —— [项目 1 项目 2 项目 3] —— 水平居中（如弹窗内容）
space-between 两端对齐，项目间间距相等（无首尾间距） [项目 1] —- [项目 2] —- [项目 3] 导航栏、均匀分布布局
space-around 项目两侧间距相等（首尾间距为项目间间距的 1/2） – [项目 1] – [项目 2] – [项目 3] – 均匀分布（需首尾留白）
space-evenly 项目间间距 + 首尾间距完全相等 – [项目 1] – [项目 2] – [项目 3] – 严格均匀分布

示例：水平居中且项目间均匀分布

.container {
  display: flex;
  justify-content: space-between; /* 两端对齐，中间间距相等 */
}

5. align-items：交叉轴对齐方式（核心）

控制项目在交叉轴方向的对齐（仅适用于「单行项目」），以 flex-direction: row 为例（交叉轴为垂直方向）：

取值 效果说明 适用场景
stretch（默认） 项目高度拉伸至与容器等高（项目无固定高度时） 等高卡片布局
flex-start 靠交叉轴起点对齐（顶部对齐） 顶部对齐的列表
flex-end 靠交叉轴终点对齐（底部对齐） 底部对齐的按钮组
center 交叉轴居中对齐（垂直居中） 水平 + 垂直居中布局
baseline 按项目内文字基线对齐（文字底部对齐） 文字对齐的卡片

示例：水平垂直居中（最常用布局）

.container {
  display: flex;
  justify-content: center; /* 主轴（水平）居中 */
  align-items: center;     /* 交叉轴（垂直）居中 */
  height: 300px; /* 容器必须有高度，否则交叉轴无拉伸空间 */
}

6. align-content：多行项目的交叉轴对齐

仅当项目换行（flex-wrap: wrap）时生效，控制「多行项目整体」在交叉轴的对齐方式（区别于 align-items 控制单个项目）：

取值 效果说明 适用场景
stretch（默认） 多行拉伸至容器高度，行间距相等 多行等高布局
flex-start 多行靠交叉轴起点对齐（顶部对齐） 多行顶部对齐
flex-end 多行靠交叉轴终点对齐（底部对齐） 多行底部对齐
center 多行交叉轴居中对齐（整体垂直居中） 多行居中布局
space-between 多行两端对齐，行间距相等 多行均匀分布
space-around 多行两侧间距相等 多行均匀分布（需留白）

示例：多行项目整体垂直居中

.container {
  display: flex;
  flex-wrap: wrap; /* 必须换行才生效 */
  align-content: center; /* 多行整体垂直居中 */
  height: 500px; /* 容器高度足够容纳多行 */
}

四、项目属性（控制单个项目）

弹性项目的属性用于单独调整某个项目的大小、对齐方式或排列顺序，覆盖容器的默认设置，共 6 个核心属性。

1. flex-grow：项目的放大比例

控制项目在容器有剩余空间时的放大比例，默认值 0（不放大）：

  • 取值为非负数字（整数 / 小数），比例越大，占有的剩余空间越多
  • 多个项目的 flex-grow 按比例分配剩余空间（如 1:2 则分别占 1/3、2/3）

示例：2 个项目按 1:2 分配剩余空间

.container {
  display: flex;
  width: 500px;
}
.item1 { flex-grow: 1; } /* 占 1/3 剩余空间 */
.item2 { flex-grow: 2; } /* 占 2/3 剩余空间 */

2. flex-shrink：项目的收缩比例

控制项目在容器空间不足时的收缩比例，默认值 1（自动收缩）：

  • 取值为非负数字，0 表示不收缩（固定宽度，避免被压缩）
  • 多个项目的 flex-shrink 按比例收缩（如 1:2 则 item1 收缩幅度是 item2 的 1/2）

示例：禁止项目收缩（常用）

.item {
  flex-shrink: 0; /* 容器空间不足时，项目不收缩 */
  width: 200px; /* 固定宽度不被压缩 */
}

3. flex-basis：项目的基准宽度

定义项目在主轴上的初始宽度（在分配剩余空间前的宽度），默认值 auto（项目自身宽度）：

  • 优先级：flex-basis > 项目自身 width（若同时设置，width 失效）
  • 可取值：px/%/em 等具体单位，或 auto（自动适应内容）

示例：设置项目初始宽度

.item {
  flex-basis: 200px; /* 初始宽度 200px，剩余空间按 flex-grow 分配 */
}

4. flex：flex-grow + flex-shrink + flex-basis 简写（常用）

为简化代码，将三个属性合并为 flex，默认值 0 1 auto，常用简写形式如下：

简写 等价于 效果说明 适用场景
flex: 1 1 1 0% 项目自动放大 / 收缩，基准宽度为 0（按比例分配空间） 等分布局、自适应宽度
flex: auto 1 1 auto 项目自动放大 / 收缩，基准宽度为自身宽度 自适应内容宽度
flex: none 0 0 auto 项目不放大、不收缩，固定自身宽度 固定宽度项目（如侧边栏）
flex: 2 0 300px 2 0 300px 放大比例 2，不收缩，基准宽度 300px 固定基准宽度的自适应项目

示例：等分布局（最常用）

.container {
  display: flex;
}
.item {
  flex: 1; /* 所有项目等比例分配容器宽度 */
  margin: 0 10px; /* 项目间距 */
}

5. align-self：单个项目的交叉轴对齐

覆盖容器的 align-items 属性，单独设置某个项目的交叉轴对齐方式，取值与 align-items 一致：

.container {
  display: flex;
  align-items: flex-start; /* 所有项目顶部对齐 */
}
.item-special {
  align-self: center; /* 该项目单独垂直居中 */
}

6. order：项目的排列顺序

控制项目的显示顺序，默认值 0，取值为整数（负数在前，正数在后，数值越小越靠前）：

.item1 { order: 2; } /* 第 2 个显示 */
.item2 { order: 1; } /* 第 1 个显示 */
.item3 { order: -1; } /* 最前面显示（数值最小） */

五、常用应用场景（附代码）

Flexbox 几乎能解决所有前端布局问题，以下是最常用的场景示例：

1. 水平垂直居中（弹窗、卡片内容）

.modal-content {
  display: flex;
  justify-content: center; /* 水平居中 */
  align-items: center;     /* 垂直居中 */
  height: 200px;
  width: 300px;
}

2. 等分布局（导航栏、选项卡）

.nav {
  display: flex;
  list-style: none;
  padding: 0;
}
.nav-item {
  flex: 1; /* 等比例分配宽度 */
  text-align: center;
  padding: 10px 0;
}

3. 两栏布局（侧边栏固定 + 主内容自适应）

.container {
  display: flex;
  height: 100vh; /* 占满视口高度 */
}
.sidebar {
  width: 200px;
  flex-shrink: 0; /* 侧边栏不收缩 */
  background: #f5f5f5;
}
.main {
  flex: 1; /* 主内容自适应剩余宽度 */
  padding: 20px;
}

4. 响应式换行布局（卡片、产品列表）

.card-container {
  display: flex;
  flex-wrap: wrap; /* 自动换行 */
  gap: 16px; /* 项目间距（替代 margin，更简洁） */
  padding: 20px;
}
.card {
  flex: 1 1 250px; /* 最小宽度 250px，超出自动换行 */
  background: #fff;
  padding: 20px;
  border-radius: 8px;
}

5. 底部固定布局（内容不足时底部贴底）

body {
  margin: 0;
  min-height: 100vh; /* 视口高度 */
  display: flex;
  flex-direction: column; /* 主轴垂直 */
}
.header {
  height: 60px;
  background: #333;
  color: #fff;
}
.content {
  flex: 1; /* 内容区占满剩余空间，推底部到底部 */
  padding: 20px;
}
.footer {
  height: 60px;
  background: #333;
  color: #fff;
  text-align: center;
  line-height: 60px;
}

6. 文字基线对齐（卡片内文字对齐）

.card-container {
  display: flex;
  align-items: baseline; /* 按文字基线对齐 */
  gap: 20px;
}
.card h3 {
  margin: 0;
  font-size: 24px;
}
.card p {
  margin: 0;
  font-size: 16px;
}

六、关键注意事项

  1. 容器高度与交叉轴对齐：align-items: center、align-content: center 等属性需要容器有明确高度（如 height、min-height: 100vh），否则交叉轴无空间，对齐效果失效。

  2. gap 属性的使用：Flexbox 支持 gap（项目间距）和 row-gap/column-gap（行列间距），无需用 margin 实现，更简洁（IE 不支持 gap，需兼容时用 margin）。

  3. **项目的 width 与 flex-basis**：flex-basis 优先级高于 width，若同时设置，width 失效；若 flex-basis: auto，则以 width 为准。

  4. 兼容性处理：IE11 支持 Flexbox，但存在部分差异（如flex-basis需配合width，gap不支持），可通过以下兼容写法修复：

    /* IE11 兼容 flex-basis */
.item {
  flex-basis: 200px;
  width: 200px; /* 兜底 IE11 */
}

  5. 嵌套 Flex 布局：弹性项目可以再声明为弹性容器，实现复杂的嵌套布局（如卡片内部的元素对齐）。
